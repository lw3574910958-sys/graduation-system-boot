# BizNotice实体类重复方法清理说明

## 清理背景
在代码审查过程中发现BizNotice实体类中存在功能重复且未使用的方法，为了提高代码质量和维护性，按照项目规范进行统一清理。

## 清理内容

### 删除的重复方法

1. **`isPublished()` 方法**
   - 功能：检查通知是否已发布
   - 删除原因：该方法在服务实现中从未被调用
   - 替代方案：使用 `canWithdraw()` 方法实现相同功能（已发布的通知才能撤回）

2. **`isDraft()` 方法**
   - 功能：检查通知是否为草稿状态
   - 删除原因：该方法在服务实现中从未被调用
   - 替代方案：使用 `canPublish()` 方法实现相同功能（草稿状态才能发布）

## 保留的核心方法

经过清理后，BizNotice实体类保留以下核心状态检查方法：

- `isEditable()` - 检查是否可以编辑
- `isFinalStatus()` - 检查是否为最终状态
- `canPublish()` - 检查是否可以发布（替代原isDraft功能）
- `canWithdraw()` - 检查是否可以撤回（替代原isPublished功能）
- `isWithdrawn()` - 检查是否已撤回
- `isSticky()` - 检查是否置顶
- `isEffective()` - 检查是否在有效期内

## 优化效果

1. **减少代码冗余**：删除了2个重复且未使用的方法
2. **提高代码一致性**：统一使用语义更清晰的方法名
3. **降低维护成本**：减少了需要维护的方法数量
4. **符合项目规范**：遵循"重复与未使用方法的统一优化"原则
